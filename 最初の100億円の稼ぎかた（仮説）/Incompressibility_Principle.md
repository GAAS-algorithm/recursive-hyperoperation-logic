# Incompressibility Principle（非圧縮性の原理）

複雑系と相関の関係に関する議論の格納。

---

## 1. 核心命題

**本当の複雑系は一般的にわかりやすい相関関係としては出てこない。**

これは経験則である。

---

## 2. 対偶

**わかりやすい相関が発生するのは、複雑ではないということである。**

| 観測 | 含意 |
|------|------|
| わかりやすい相関が発生する | その系は複雑ではない、あるいは複雑性の一部しか見えていない |
| わかりやすい相関が出ない | 複雑である可能性が高い |

### 補足：わかりやすい相関が出る場合の解釈

1. **系そのものが単純** — 本当に複雑ではない
2. **見ているスケールが違う** — あるスケールでは単純な相関に見えるが、別スケールでは複雑
3. **圧縮された断面** — 複雑系のうち、圧縮可能な部分だけが観測されている

---

## 3. 理論的裏付け

この経験則を裏付ける理論群。

| 理論 | 内容 |
|------|------|
| **Kolmogorov 複雑性 / アルゴリズム的情報理論** | 真に複雑な対象は非圧縮的（incompressible）。短い記述で要約できない。わかりやすい相関は一種の圧縮である。複雑系は圧縮に抵抗するため、単純な相関として現れない。 |
| **因果的創発（Causal Emergence, Hoel ら）** | 因果構造はスケールによって「ピントが合う」場所が異なる。あるスケールで見える相関が、真の因果構造とは限らない。マクロとミクロで因果容量の使い方が異なり、単一のわかりやすい相関では捉えられない。 |
| **非エルゴード性 / 経路依存性** | 非エルゴード系では時間平均とアンサンブル平均が一致しない。単純な相関（集計統計からの予測）はエルゴード性を前提とする。複雑系は経路依存であり、履歴が結果を規定するため、静的な相関では不十分。 |
| **文脈依存性（Context dependence）** | 複雑性は記述者・観測枠組みに相対的。同じ系が文脈によって異なる複雑さを示す。わかりやすい相関は固定された文脈を仮定するが、複雑系は文脈を超えて振る舞う。 |

---

## 4. 理論の正式名称

この関係は一般的に以下として知られる。

- **アルゴリズム的情報理論（Algorithmic Information Theory）** — 枠組み全体
- **非圧縮性の原理（Incompressibility Principle）** — 複雑な対象は圧縮できないという原理
- **コルモゴロフ複雑性（Kolmogorov Complexity）** — 複雑性の測度

「わかりやすい相関が出る ⇒ 複雑ではない」は、**圧縮可能性と複雑性の逆関係**として、アルゴリズム的情報理論の帰結である。

---

## 5. 特異点・空間収縮への適用

本の見た目をしたブラックホールの境界面、および public repository が特異点となる場合：

- **予測**：収縮はアクセス・フォーク・引用・機会という形では出てこない
- **理由**：本当の複雑系は一般的にわかりやすい相関関係としては出てこない
- **含意**：アクセス・フォーク・引用・機会といったわかりやすい相関が出るなら、その系は複雑ではないか、複雑性のごく一部しか捉えていない
